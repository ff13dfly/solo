# AI 辅助产品信息提取协议 (Extraction Protocol)

> [!NOTE]
> 本协议的提取能力已实现,但调用方法名以代码为准:使用 `agent.image.parse`(`mode='product'`),非旧文档的 `agent.image.product`。

## 1. 目标概述
本协议定义了利用 AI 的视觉识别（Vision）和多语言处理能力，通过产品实拍照片自动提取、翻译并更新产品信息的标准化流程。

### 核心设计原则
- **无状态 AI 调用**：前端传递预处理后的 Base64 图像，Agent 仅负责实时解析，不存储中间状态。
- **前端暂存 (Staging)**：识别结果缓存在前端预览区，由用户人工确认。
- **类目驱动 (Category-Driven)**：提取规则（Schema）由类目元数据动态下发。
- **松耦合上下文**：通过通用的 `meta` 参数传递业务相关的非结构化上下文。

---

## 2. 交互流程

### 第一阶段：规则下发 (Discovery)
- **Schema 获取**：前端从所属分类的 `meta.ai_extraction_schema` 获取字段定义。
- **Meta 准备**：前端根据业务场景准备扩展元数据（如图片角色：正面、铭牌等）。

### 第二阶段：采集与压缩 (Ingestion)
- **客户端预压缩**：利用 Canvas 将长边限制在 1200px 左右，转换为 JPEG (0.7-0.8 质量)。
- **角色标记**：为每张图片附加语义角色，通过 API 的 `meta` 参数透传。

### 第三阶段：AI 语义解析 (Analysis)
- **接口调用**：发送 `Base64[] + Schema + Meta` 至 `agent.image.parse`（产品结构化提取通过 `mode='product'` 参数启用）。
- **解析逻辑**：
    - **跨图整合**：AI 自动整合多张照片的上下文。
    - **置信度计算**：为每个字段返回 0-100 的 `confidence`。
    - **自动裁剪**：返回 `bounding_box` 坐标。

### 第四阶段：本地化校验 (Localization & Verification)
- **事实溯源**：区分内容是“原生提取 (Extracted)”还是“AI 翻译 (Translated)”。
- **标记机制**：纯 AI 翻译内容默认附加 `pending_verify` 状态，需人工二次核对。
- **去重合并**：多图冗余信息由 AI 在语义层完成去重。

### 第五阶段：确认与提交 (Commit)
- **本地持久化**：中间结果暂存在 LocalStorage/IndexedDB 以支持 Session 恢复。
- **正式更新**：用户确认后，同步数据至商品库并清理本地暂存。

---

## 3. 技术优势
1. **高性能**：前端预压缩将响应耗时控制在 5s 以内。
2. **高灵活性**：松耦合的 `meta` 接口支持适配任意业务场景。
3. **数据可审计**：通过置信度与溯源标记确保自动化录入的质量可控。
